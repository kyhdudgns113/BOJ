## **설명**

<div class="block_angle_60">

### **입력**

- 1부터 30까지의 자연수중 랜덤하게 중복없이 28개가 한 줄 씩

</div>

<div class="block_angle_90">

### **상황**

- 30명중 과제를 낸 사람이 28명이 있다.

</div>

<div class="block_angle_120">

### **출력**

- 과제를 안 낸 사람들중 숫자 낮은것부터 한 줄 씩 출력한다.

</div>

## **소스 코드**

<div class="block_green">

### **Node**

``` js
const fs = require('fs')
const inputs = fs.readFileSync('/dev/stdin').toString().split('\n').map(Number)

for (let i = 1; i <= 30; i++) {
  if (!inputs.includes(i)) {
    process.stdout.write(`${i}\n`)
  }
}
```

<br />

- #### **저 코드대로면 이중 반복문을 도는 꼴이라 느려진다**

    + 하지만 데이터는 많아봐야 30개다

    + 이중 반복문을 돌아도 속도에 전혀 지장이 없다.

    + 그러면 빠르게 작성하는 코드, 이해하기 쉬운 코드가 더 좋을 수 있다.

    + 이중 반복문 안 쓰는 코드는 python 에서 짜놨다.

- #### **정렬 과정이 필요하지 않다**

    + 숫자 낮은것부터 배열에 없는지 확인을 하기 때문이다.

- #### **대용량 입출력을 연습하기 위해 process.stdout.write 함수를 쓰는 습관을 들인다.**

</div>

<div class="block_blue">

### **Python**

``` python
import sys

input = sys.stdin.readline
output = sys.stdout.write

studentArr = [i + 1 for i in range(30)]

for i in range(28):
  student = int(input())
  studentArr[student - 1] = 0

for s in studentArr:
  if s > 0:
    output(f"{s} ")
```

<br />

- #### **학생 번호 배열을 하나 만들고, 과제 제출한 학생은 번호를 0으로 만들었다**

    + 이후 번호가 남아있는 학생은 과제를 제출하지 않은것이다.

    + 그 학생들만 출력하면 된다.

    + 번호가 낮은 순서부터 확인하기에 정렬과정이 필요하지 않다.

- #### **Python 역시 대량 입출력에 익숙해지기 위해 저 함수들을 쓴다.**

- #### **output 은 format 을 지정하여 커스터마이징이 용이하도록 한다.**

    + C 스타일의 잔재가 안쓴지 수년이 지나도 남아있다.
