## **소스 코드**

<div class="block_green">

### **Node**

``` js
const fs = require('fs')
const inputs = fs.readFileSync('/dev/stdin').toString().split('\n')

const [N, M] = inputs[0].split(' ').map(Number)

const basketArr = Array(N).fill(0)

for (let idx = 0; idx < M; idx++) {
  const [i, j, k] = inputs[idx + 1].split(' ').map(Number)

  for (let bIdx = i - 1; bIdx < j; bIdx++) {
    basketArr[bIdx] = k
  }
}

process.stdout.write(`${basketArr.join(' ')}`)
```

<br />

- #### **splice 함수를 써도 되긴 한다**

    + 코드 직관성도 떨어지고, 성능도 감소한다.

- #### **대용량 입출력을 연습하기 위해 process.stdout.write 함수를 쓰는 습관을 들인다.**

### **Python**

``` python
import sys

input = sys.stdin.readline
output = sys.stdout.write

N, M = map(int, input().split())

basketArr = [0 for i in range(N)]

for idx in range(M):
  i, j, k = map(int, input().split())

  basketArr[i:j + 1] = [k] * (j - i + 1)

output(f"{" ".join(str(basketArr[i]) for i in range(N))}")
```

<br />

- #### **이중 for 문을 써도 된다.**

- #### **저 방식이 속도가 훨씬 빠르다**

    + 이중 for 문은 Interpreter 형식으로 작동함

    + slice 는 C 코드로 작동함

- #### **Python 역시 대량 입출력에 익숙해지기 위해 저 함수들을 쓴다.**

- #### **output 은 format 을 지정하여 커스터마이징이 용이하도록 한다.**

    + C 스타일의 잔재가 안쓴지 수년이 지나도 남아있다.
