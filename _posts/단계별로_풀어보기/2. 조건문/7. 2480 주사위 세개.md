## **설명**

- 첫쨰줄에 주사위값 A, B, C 를 입력받는다.

- 3개가 모두 같으면 10,000 + 1,000 * A

- 2개가 같으면 1,000 + 100 * 같은수

- 다 다르면 100 * 제일큰수

</div>

<div class="block_grey">

## **소스 코드**

<div class="block_green_transparent">

### **Node**

``` js
const fs = require('fs')
const inputs = fs.readFileSync('/dev/stdin').toString()

const [a, b, c] = inputs.split(' ')

const [A, B, C] = [parseInt(a), parseInt(b), parseInt(c)].sort()

let score = 0

if (A === B && B === C) {
  score = 10000 + A * 1000
} // ::
else if (A === B || B === C) {
  score = 1000 + B * 100
} // ::
else {
  score = C * 100
}

process.stdout.write(`${score}`)
```

<br />

- #### A, B, C 를 정렬하면 적은 조건문으로도 풀 수 있다.

- #### 두 번째 조건문에서, B 는 어차피 가운데 수라 둘 중 하나와 같다.

    + B 같에 대해서 계산하면 된다.

- #### 나중에 대량 입출력이 필요할때를 대비하여 저런 입출력을 사용한다.

- #### 충분히 더 짧은 코드를 작성할 수 있으나, 가독성을 위해 저렇게 짰다.

    + 어차피 실제로 작업할때는 잘 쓰지 않는다.

    + 불필요한 잔기술인 경우가 종종 있다.

    + <s>옛날에 이미 많이 해봤다</s>

### **Python**

``` python
import sys

input = sys.stdin.readline
output = sys.stdout.write

sortedArr = list(map(int, input().split()))
sortedArr.sort()

A, B, C = sortedArr

score = 0

if A == B and B == C:
  score = 10000 + 1000 * A
elif A == B or B == C:
  score = 1000 + 100 * B
else:
  score = 100 * C

output(f"{score}")
```

<br />

- #### Python 도 같은 방식으로 풀었다.

- #### Python 역시 대량 입출력에 익숙해지기 위해 저 함수들을 쓴다.

- #### output 은 format 을 지정하여 커스터마이징이 용이하도록 한다.

    + C 스타일의 잔재가 안쓴지 수년이 지나도 남아있다.
