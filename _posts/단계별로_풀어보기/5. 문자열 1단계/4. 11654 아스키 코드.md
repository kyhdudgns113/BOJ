## **소스 코드**

<div class="block_green">

### **Node**

``` js
const fs = require('fs')

const input = fs.readFileSync('/dev/stdin').toString().trim()

// process.stdout.write(`${input.charCodeAt(0)}`)

const tEncoder = new TextEncoder()
const u8Arr = tEncoder.encode(input)

process.stdout.write(`${u8Arr[0]}`)
```

<br />

<div class="block_white tp_80">

### **공통**

- #### 리눅스 환경에서 이상한 문자가 추가되어 문자열에 들어가는것을 방지하고자 trim 을 썼다.

- #### 대용량 입출력을 연습하기 위해 process.stdout.write 함수를 쓰는 습관을 들인다.

</div>

<div class="block_grey tp_80">

### **특이점**

- #### charCodeAt 을 쓰면 해당 character 의 아스키나 UTF-16 코드를 알 수 있긴 하다.

    + 하지만 그러면 재미도 없고, 공부도 안된다.

- #### ByteArr 형태로 바꾼 뒤, 그 값을 출력했다.

    + JS 는 이걸 알아서 십진수 숫자로 바꿔준다.

</div>

<div class="block_blue">

### **Python**

``` python
import sys

input = sys.stdin.readline
output = sys.stdout.write

C = input().strip()

output(f"{ord(C)}")
```

<br />

<div class="block_white tp_80">

### **공통**

- #### Python 역시 대량 입출력에 익숙해지기 위해 저 함수들을 쓴다.

- #### output 은 format 을 지정하여 커스터마이징이 용이하도록 한다.

</div>
